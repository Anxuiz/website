Technical Details

The portfolio is a single page application (SPA). That means the entire "website" is actually just a single web page that uses JavaScript to change the contents and provide interactivity. For HTML/CSS sanity I use the Bootstrap framework and for JavaScript sanity I use AngularJS. Note the use of "sanity" -- I don't particularly like web development, but these two tools make it bearable.

I try to describe as much of the site using JSON objects and HTML templates to reduce boilerplate and put the data into more structured form. For example, I have a service that has all my classes in JSON format so the schedule page can generate a pretty looking schedule on the fly.

For hosting I use Github Pages because it's free and stable. While it does not support dynamic pages, I don't think it will become a problem because the portfolio is very simple. Also, fortunately it supports custom domains so I can get a very professional looking website address. Yay!
